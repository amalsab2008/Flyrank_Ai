## 1. Typography Selection (Max 2 Fonts)

To maintain a clean, intentional design across the portfolio, exactly two free Google Fonts have been chosen:

| Role | Font Name | Style / Usage | Source Link |
| :--- | :--- | :--- | :--- |
| **Heading Font** | **Outfit** | Bold, geometric, sharp headers (`h1`, `h2`, `h3`) | [Google Fonts - Outfit](https://fonts.google.com/specimen/Outfit) |
| **Body Font** | **Inter** | Highly legible, neutral, ultra-clean UI body copy | [Google Fonts - Inter](https://fonts.google.com/specimen/Inter) |

*Rationale: `Outfit` provides modern technical character for headings, while `Inter` delivers maximum legibility for dense code snippets, case study text, and technical breakdowns.*

## 2. Tight Color Palette (4 Colors + Hex Codes)

The color palette is deliberately restricted to 4 calm, high-contrast tones so that case study metrics and code artifacts remain the loudest elements on the page.

| Layer / Role | Hex Code | Visual Sample | Description / Usage |
| :--- | :--- | :--- | :--- |
| **Primary Background** | `#0F172A` | `[ Slate 900 ]` | Deep slate near-black background for zero glare |
| **Surface / Card** | `#1E293B` | `[ Slate 800 ]` | Subtle container background for case study cards & code blocks |
| **Primary Text** | `#F8FAFC` | `[ Slate 50 ]` | Crisp near-white for high legibility text and headings |
| **Primary Accent** | `#6366F1` | `[ Indigo 500 ]` | Calm, modern technical accent for buttons, borders, and active links |
