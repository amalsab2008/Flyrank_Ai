## 2. Real Build Log & Iteration History

Documenting real build iterations, technical roadblocks, and design deviations from the FL-06 specification:

### Iteration 1: The GSC/GA4 Direct Query Join Failure
* **What Happened**: In the initial draft script, I attempted to join GSC Site Queries directly with GA4 event records on a hypothetical `query` field.
* **Why It Broke**: GA4 does not store organic search query dimensions due to user privacy features. The join returned 0 matching rows.
* **Course Correction**: Updated the join architecture to align GSC URL impressions with GA4 event exports strictly on `landing_page_url`. This successfully mapped search demand to conversion revenue.

### Iteration 2: Anonymized Query Handling
* **What Happened**: GSC URL impressions contained ~36% anonymized query rows (`query == ""`). Passing empty strings to the semantic intent classifier caused `KeyError` exceptions.
* **Course Correction**: Added an explicit guardrail inside `fetch_striking_distance_keywords()` to skip blank queries during query-level intent modeling while retaining impression metrics during URL aggregation.

### Spec Deviations & Cuts:
* **Cut Heavy Embeddings Model**: Rather than requiring a heavy 500MB `sentence-transformers` download, I implemented a zero-dependency rule-based intent classifier. This reduced execution time from 12 seconds to 0.04 seconds with zero API cost.
