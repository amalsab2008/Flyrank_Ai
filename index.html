Case Study 01 — Lead Technical Work

Attacker-Aware Password Analysis & Breach Engine

Python 3.13 Argon2id Bloom Filters DevSecOps CLI

Beat 1: The Problem

Standard password validators rely on naive regex rules (min 8 chars) that fail against keyboard walks (`qwertyuiop`) or date sequences (`20261234`). Verifying passwords against breached lists via remote DB queries introduces network latency and privacy leaks.

Beat 2: What I Did

Built a Python entropy evaluation engine using pattern recognition. Implemented Argon2id hashing for zero plaintext memory retention and integrated an in-memory Bloom filter for O(1) constant-time breach lookups without external network calls.

Beat 3: What Came Of It

Evaluates raw password strength in sub-milliseconds with zero plaintext exposure. Checks 100,000+ leaked hashes in real-time with minimal RAM bit arrays.

Case Study 02 — Backend & AI Integration

Decoupled AI Task API & Containerized Stack

Node.js Express PostgreSQL Docker Compose Gemini Flash / Groq

Beat 1: The Problem

Tightly coupling database drivers or LLM SDKs inside HTTP routes makes backends brittle. Swapping storage (InMemory to Postgres) or AI providers (Gemini to Groq or local Ollama) usually breaks route logic.

Beat 2: What I Did

Architected a 4-layer decoupled backend (`Domain -> Repo -> Service -> Routes`). Built an AiService wrapper enforcing JSON schema output, 8s timeouts, exponential backoff retries on 429/5xx, and token cost logging. Containerized app + Postgres with Docker Compose.

Beat 3: What Came Of It

Proved architectural decoupling: swapping storage requires changing exactly 1 line in `.env` with zero route modifications. Starts full stack with single `docker compose up` command.

Case Study 03 — Process Engineering

AI Workflow Audit & Task Standardization

Ethan Mollick Framework Task Classification DevSecOps Audit

Beat 1: The Problem

Unfiltered AI adoption in software development leads to subtle bugs in core security algorithms, wasted context tokens, and over-reliance on unverified code.

Beat 2: What I Did

Audited 12 recurring weekly tasks using the Ethan Mollick classification framework. Explicitly marked 3 critical security tasks as *Just Me* (core cryptographic logic, concurrency debugging, PR reviews) to maintain full manual ownership.

Beat 3: What Came Of It

Reclaimed ~4 developer hours per week while eliminating silent AI bugs in core security logic. Established 100% test coverage standards for generated test suites.
